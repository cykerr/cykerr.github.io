How C++ Works

How the C++ Compiler Works

c++ 编译器唯一要做的就是把源码变为中继格式 —— obj

具体步骤：

  1. compiler 需要预处理（pre-process）代码
  2. 预处理后，进入标记解释（tokenizing）和解析（parsing）阶段
  3. 创建抽象语法树（abstract syntax tree）来进行代码表达

当编译器创建了这颗抽象语法树后，就可以开始产生代码了，这个代码才是真正的cpu会执行的机器码，总之，编译器的工作就是把代码转换成常数资料（constant data）或者指令（instructions）。

How the C++ Linker Works

compiler 将每个c++源文件编译成独立的 obj 文件，linker将这些obj文件中的函数和符号链接在一起，形成可执行的二进制文件。

头文件

在一个文件中创建函数，然后想要在另一个文件中使用时，如果不添加该函数的声明，编译器将无法识别。所以需要一个共同的地方来存放声明，而非定义（函数只能被定义一次），这个地方就是头文件，它的作用只是告诉编译器，“嘿，这个函数存在！”。

指针

指针是一个整数，它存储着一个内存地址。

引用

引用是指对某个已存在的变量的引用（可看作别名），引用本身并不是一个新的变量，不占用内存。引用本质上是基于指针的一种高级修饰（一种特殊的指针），使得代码更易读更好学而已。

类与结构体

  • class 代码块内部如果不加访问修饰符，默认是私有的。
  • struct 代码块内部如果不加访问修饰符，默认是公有的。

struct 与 class 的区别仅限于此，struct 存在的原因是 c++ 要与 c 保持兼容性，而 c 中没有 class。
